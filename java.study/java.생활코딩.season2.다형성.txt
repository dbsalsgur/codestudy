1. 메소드와 다형성
- 다형성(polymorphism) : 하나의 메소드나 클래스가 있을 떄, 이것들이 다양한 방법으로 동작하는 것
- 다형성의 가장 쉬운 예가 overloading이다.
- 한 클래스에 다른 데이터타입을 가진 같은 이름의 메소드를 만든다면, 입력 값의 데이터 타입에 따라 실행되는 메소드가 다를 것이다.

2. 클래스와 다형성
- 클래스 B가 클래스 A를 상속한다면, 데이터타입이 A인 인스턴스 B를 생성할 수 있다.
ex) A obj = new B();
  → 클래스를 인스턴스화 시킬 때, 데이터 타입은 자기 자신 또는 자신의 부모 클래스가 될 수 있다.
- 이때, 데이터타입인 A 클래스에 있는 메소드만 호출이 가능하다.
- 만약 A클래스에 있는 메소드x()를 B클래스에 overriding했다고 하자.
- 이떄, A obj = new B를 해서, x()라는 메소드를 호출하면,  B에 있는 overriding된 메소드가 호출된다.
  → A 클래스에 정의되어 있는 메소드기 때문에 호출 가능하다.
  → 오버라이딩 된 메소드를 우선적으로 호출한다.
  → B 클래스에만 정의되어 있는 메소드는 호출 불가하다.
